Ollama + Open WebUI 로컬 AI 챗봇 구축기

calendar_today 2026.02.18 visibility 1,842
#AI #로컬LLM #Ollama

왜 로컬 LLM인가?

ChatGPT, Claude 같은 클라우드 기반 LLM은 편리하지만, 비용, 프라이버시, 오프라인 사용 측면에서 제약이 있습니다. 특히 회사 내부 코드를 다루거나, API 비용이 부담되는 개인 프로젝트에서는 로컬에서 직접 모델을 돌리는 것이 훨씬 합리적입니다.

"내 GPU에서 돌리면 무료인데, 왜 매달 $20을 내야 하지?" — 이 단순한 의문에서 시작했습니다.

환경 & 준비물

이번 구축에 사용한 환경은 다음과 같습니다:

  • GPU: NVIDIA RTX 4060 Ti 16GB
  • OS: Ubuntu 22.04 LTS
  • RAM: 32GB DDR5
  • Docker: 24.0.7

VRAM이 16GB이므로 7B~13B 파라미터 급 모델을 양자화해서 쾌적하게 돌릴 수 있습니다. 8GB VRAM이라면 7B Q4 양자화 모델을 권장합니다.

Ollama 설치

Ollama는 원커맨드로 설치할 수 있습니다:

bashcurl -fsSL https://ollama.ai/install.sh | sh

# 설치 확인
ollama --version
# ollama version 0.3.14

설치 후 ollama serve 명령으로 서버를 시작하면, 기본적으로 http://localhost:11434에서 API가 열립니다.

모델 선택과 다운로드

Ollama의 모델 라이브러리에서 용도에 맞는 모델을 선택합니다. 제가 테스트한 모델별 성능 비교입니다: 

bash# 범용 대화 (추천)
ollama pull llama3.1:8b-instruct-q5_K_M

# 코딩 특화
ollama pull codellama:13b-instruct-q4_K_M

# 한국어 특화
ollama pull qwen2.5:7b-instruct-q5_K_M

양자화와 VRAM

양자화(Quantization)는 모델의 가중치를 FP16 → INT4/INT8로 줄여 메모리 사용량을 대폭 줄이는 기법입니다. 체감 품질 하락은 미미합니다.

Open WebUI 연동

Ollama API만으로도 사용할 수 있지만, Open WebUI를 연결하면 ChatGPT와 동일한 채팅 인터페이스를 얻을 수 있습니다. 

bashdocker run -d -p 3000:8080 \
    --add-host=host.docker.internal:host-gateway \
    -e OLLAMA_BASE_URL=http://host.docker.internal:11434 \
    -v open-webui:/app/backend/data \
    --name open-webui \
    --restart always \
    ghcr.io/open-webui/open-webui:main

http://localhost:3000에 접속하면 ChatGPT와 거의 동일한 UI가 나타납니다. 대화 기록 저장, 시스템 프롬프트 설정, RAG까지 무료로 사용 가능합니다.

성능 튜닝 팁

  1. GPU 레이어 수 조정: num_gpu 파라미터로 GPU에 올릴 레이어 수를 조정
  2. 컨텍스트 윈도우: num_ctx를 2048에서 4096으로 늘리면 긴 대화에 유리
  3. 배치 사이즈: num_batch를 512로 설정하면 처리 속도 향상
  4. Keep Alive: keep_alive를 -1로 설정하면 모델이 메모리에 상주

마무리

RTX 4060 Ti 한 장으로도 충분히 실용적인 로컬 AI 챗봇을 구축할 수 있었습니다. 월 $0으로 프라이빗한 AI 어시스턴트를 운영할 수 있다는 것은 큰 장점입니다.

다음 글에서는 RAG(Retrieval-Augmented Generation)를 연동하여 내 문서를 기반으로 답변하는 챗봇을 만드는 방법을 다루겠습니다.
